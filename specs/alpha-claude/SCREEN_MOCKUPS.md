## Key UI/UX Principles

1. **Misk UI Conformance**: Use Misk component library, color scheme, typography
2. **Accessibility**: ARIA labels, keyboard navigation, focus management
3. **Mobile-Responsive**: Adaptive layouts (not in v1 scope but design should allow)
4. **Performance**: Lazy loading, infinite scroll, optimistic UI updates
5. **Feedback**: Loading states, success/error toasts, inline validation
6. **Markdown**: Safe rendering (no XSS), syntax highlighting for code blocks
7. **Progressive Enhancement**: Core functionality works without JS (where feasible)

---

## Component Reuse Patterns

**Post Card** (reusable across feeds, profiles, search):
- Upvote button + count
- Title (clickable)
- Metadata (author, timestamp, tags)
- Actions (comment, share, edit, delete)
- Optional: summary, badges, read indicator

**Comment Tree** (recursive):
- Author, timestamp, upvote
- Body (rendered markdown)
- Actions (reply, edit, delete)
- Children (nested, indented)

**Tag Chip** (reusable):
- Tag name
- Clickable to filter
- Removable (in edit contexts)

**User Avatar + Name**:
- Avatar image
- Username (clickable to profile)
- Optional: points badge, follow button

---

These mockups should guide UI implementation for each commit. LLM builders should follow these patterns while adapting to Misk UI specifics.
